Content

一个URI由以下可选部分组成。

mklink

创建软连接

Syntax
    mklink /d link_name linked_absolute_path_or_filename

常用的协议(scheme_name)有：

  • /d: 连接到目录标识
  • http: HTTP协议
  • ftp: FTP协议
  • sqlite: SQLITE数据库

driver

Windows驱动器盘符

Syntax
    driver ::= [a-zA-Z]:

Letter为a-z或者A-Z的字符。在Windows下缺省表示当前盘符。

root

根目录

Syntax
    root ::= [\/]

Linux下为'/'，Windows下为'\'。缺省表示相对当前目录的相对路径(见direction小节)。

direction

目录

Syntax
    direction ::= ([\w .]+[/]?)+

特别的目录标识

  • '.': 当前目录
  • '..': 父目录
  • '...': 祖目录

file

文件

Syntax
    file ::= [\w .]+

ext

文件扩展

Syntax
    ext ::= [\w]+

query

参数

Syntax
query ::= ?query-form&query-form...
query-form ::= query-key[=query-jsonlike-value]
  • query-jsonlink-value: JSON格式的查询值。若忽略，则query-key为Boolean变量，且其值为True
