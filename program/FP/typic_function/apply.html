Background

OOP以数据为设计核心，所有的处理（OOP称之为方法）都绑定到了数据类型中。若需要为当前类型增加新的处理，按照OOP的设计，应该扩展一个子类，在子类中添加新的方法，然后在原使用父类的地方都使用子类来替代。

这种扩展的问题在于：

返回值类型

动态语言不需要考虑返回值类型。但静态语言每个值（包括返回值）必须有类型。

返回方式

隐式返回

以函数体内最后一个语句的返回值作为函数的运算结果返回。

显示返回

使用return语句显示返回结果。

Syntax

return [value-expression];
  • return: 关键字
  • value-expression: 返回值。忽略表示返回void

return返回可以在函数体内的任意位置。它提供了一种在任意位置直接跳出函数的方法。在一些场景下可以简化代码的层级，例如：

func test(){
  if(...) return -1;
  ...
}

多值返回

函数同时返回多个值。

一般使用返回一个可以承载多个数据的集合类型的方法。若返回的集合类型支持destructure，则从使用这角度看，表现上就是同时返回了多个值。
